# Documentation guide

Start with [`ARCHITECTURE.md`](ARCHITECTURE.md) for the current repository layout and runtime load
order.

The other documents are design records, audits, and implementation briefs written at different
points in the prototype's history. Their bare filenames and line numbers may describe the former
flat root layout; use the ownership map in `ARCHITECTURE.md` to find each file's current home under
`src/`.

- `WDYM_DESIGN.md` — WHO? DID YOU MAKE? rules and design intent
- `HAIR_COMPOSITOR.md` — hair composition model and diagnostics
- `GROUPTHINK_QUALITY_REPORT.md` — Groupthink rules analysis
- `face-generator.md` — portrait generator notes
- `8-player-brief/` and `refactor-brief/` — historical implementation packets, not current setup
